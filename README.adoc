|An implementation of the **Model Context Protocol (MCP)** that gives AI agents a reliable mathematical expression evaluator. Built with Rust for speed, correctness, and a minimal footprint.

Unlike using the LLM's own arithmetic, this server computes results **exactly** using a dedicated expression engine -- no hallucinated digits, no floating-point approximation surprises.
|===

toc::[]

NOTE: This server uses **stdio** transport, which is the standard for local MCP servers launched directly by the AI client as a child process.

== Why This Exists

LLMs are notoriously unreliable for arithmetic: rounding errors, wrong operator precedence, silent mistakes on large numbers or complex expressions. This server offloads all calculations to a deterministic Rust engine.

== Features

* **Expression evaluation**: Arithmetic, powers, modulo, parentheses.
* **27 built-in functions**: `sin`, `cos`, `tan`, `sqrt`, `log`, `abs`, `ceil`, `floor`, `round`, and more.
* **Named variables**: Pass `x`, `y`, or any identifier alongside the expression.
* **Built-in constants**: `pi` and `e` available out of the box.
* **Compilation cache**: Repeated calls with the same expression are served from cache (~15–80 ns/eval).
* **Structured logging**: Configurable via `MCP_CALC_LOG` environment variable, output on stderr (never pollutes the stdio transport).
* **Minimal dependencies**: Only `serde_json`, `mathexpr`, and `tracing`. No async runtime needed.

== Tool

The server exposes a single tool:

[cols="1,3"]
|===
| Tool | Description

| `evaluate`
| Parses and evaluates a mathematical expression string. Supports named variables passed as a JSON object.
|===

=== Example Calls

Simple constant expression:
[source,json]
----
{
  "name": "evaluate",
  "arguments": {
    "expression": "2 * pi * 6371"
  }
}
----

With named variables:
[source,json]
----
{
  "name": "evaluate",
  "arguments": {
    "expression": "sqrt(x^2 + y^2)",
    "vars": { "x": 3.0, "y": 4.0 }
  }
}
----

Result: `5.0`

== Building

Requires Rust 1.85+ (Edition 2024).

[source,bash]
----
cargo build --release
----

The binary will be at `./target/release/mcp-math`.

== Configuration

[cols="1,3,1"]
|===
| Variable | Description | Default

| `MCP_CALC_LOG`
| Log level (`debug`, `info`, `warn`, `error`)
| `info`
|===

Setting `MCP_CALC_LOG=debug` enables cache hit/miss traces and variable binding details -- useful during development.

== Connecting to an AI Client

Add the server to your MCP client configuration. Example for **LM Studio** (`mcp.json`):

* **Location**:
** Windows: `%APPDATA%\LM Studio\mcp.json`
** macOS: `~/Library/Application Support/LM Studio/mcp.json`
** Linux: `~/.config/lmstudio/mcp.json`

[source,json]
----
{
  "mcpServers": {
    "math": {
      "url": "http://YOUR_HOST:3000/sse"
    }
  }
}
----

TIP: Use the **release build** in production. The debug build is significantly slower for expression compilation.

        └── mod.rs
----

== Known Limitations

* **No variable type coercion**: All variables must be JSON numbers (`float`). Strings are silently ignored.
* **Cache key includes variable names**: The same expression compiled with different variable sets creates separate cache entries. This is intentional for correctness.
* **No JavaScript / symbolic math**: This is a numerical evaluator only. Symbolic derivation or simplification is out of scope.
* **Single-threaded**: stdio transport is inherently sequential; one request is processed at a time.
